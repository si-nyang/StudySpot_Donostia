Home Page
Description: This function generates the home page of the Study Spots website. It acts as the main entry point of the system and provides navigation to all other features such as the list page, map page, add location page, and edit location page.
The page is dynamically generated using a Java servlet and displays interactive cards created using JavaScript.
Use:
First, the user opens the website or clicks the home page link.
Then, the servlet generates the main menu and displays all available navigation cards.
The user can click on any card to navigate to the corresponding feature page.
menu.java This servlet generates the HTML structure of the home page, including the header and an empty container for cards that are filled dynamically using JavaScript.
script.js This file creates and displays navigation cards dynamically. Each card contains a title, description, explanation, test data, and a link to a specific page of the application.

List Page
Description: This function lists all of the locations inside the database. It displays name, average rating, category, address, image, and a link to the detail page.
The servelet also allows users to filter the locations by category, University, Cafe, Library, or All.
Use:
First, the user scrolls through all available locations. The user can then filter in order to find what they're looking for. Once found they can either click and go to the detail page for that location or return to the home page.
LocationList.java HTML code for the servlet which retrieves the data using LocationData.java and displays it within a table. It checks whether there is a category parameter, if there is it only retrieves the information from that category
LocationData.java Executes sql and retrives/updates data from the database, in this case it retrieves a vector of locations

Detail Page
Description: This function shows detailed information about a selected study spot. It displays the name, creator, rating, category, description, address, hours, tags, and image.
The servlet retrieves the data from the database using the location id.
Use:
First, the user selects a location from the list page.
Then, the detail page displays all information about that location.
LocationDetail.java Retrieves the selected location from the database and generates the detail page.
LocationData.java Handles database queries for retrieving location data.
Map Page
Description: This function shows all study spots on an interactive map using markers.
The page requests data from the server and displays each location on the map.
Use:
First, the user opens the map page.
Then, markers appear for each study spot, and clicking a marker shows its information.
Map.html Displays the map and creates markers using JavaScript.
mapJson.java Returns location data from the database in JSON format.

Chinatsu Oki
Add Location Page
Description: This function allows the user to add a new study spot by filling in a form.
The Add Location page contains a form where the user enters information. When the form is submitted, the addLocation servlet reads the input values, combines the selected tags into one string, creates a LocationData object, inserts it into the Locations table, and then redirects the user to the home page.
Use:
First, the page automatically gets username using session.
Then, the user imputs information about a new study spot which is category, name, description, address, hours, tags, and rating.
Once the user submits the form, the page redirects to the homepage.
Session.java This file is a servlet where the user sees and inputs information.
addLocation.java This file reads and structures the data of the form submitted, sends it to LocationData servlet, then redirects to the homepage.
LocationData.java (only InsertLocation) This part receives data from addLocation.java and inserts it into the database.

Edit Location Page
Description: This function allows users to modify existing study spot information. It retrieves current data from the database, displays it in a form, and updates the record upon submission.
The page ensures that any changes to a spot's details are saved and reflected across the system.
Use:
First, the user navigates to the edit page (usually from a detail view).
Then, they change the desired information in the input fields and click "Speichern".
The system updates the database and redirects the user back to the updated location details.
UpdateLocation.java This servlet handles both displaying the edit form (GET) and processing the database update (POST).
LocationData.java (updateLocation) This method executes the SQL UPDATE statement to save the modified location data into the database.
